# 验收标准文档 — 风暴中的大顺朝

## 验收规则

1. 每项验收标准必须可对照、可判定
2. 可量化指标给出具体数值
3. 非量化场景明确写出判定方法
4. 开发完成后必须提供每项验收通过的证据
5. 验收项100%覆盖规格文档中的所有交付内容

## 验收流程

1. 验收代理逐条执行 AC-01 至 AC-26
2. 每项标注：✅通过 / ❌不通过
3. 不通过项：记录失败原因 + 对应约束文档位置
4. 全部通过 → 交付完成
5. 存在不通过项 → 返回开发代理修复 → 重新验收
